SLIP Dungeon RPG

SLIP S-expression 언어로 구현한 5층 던전 RPG. 클래스 기반 아키텍처, 인터랙티브 REPL 전투 시스템.

실행 python -X utf8 src/factory_shell.py play demo/game.cf

빠른 시작

Manual

게임을 로드한 뒤 REPL에서 아래 커맨드를 입력합니다. (start_game)으로 시작, 각 턴에 행동을 선택하세요.

게임 흐름 요약

① (start_game) 입력 → ② 층별 몬스터와 교대 전투 → ③ 5층 보스 드래곤 군주 격파 → 클리어

전투는 매 턴 플레이어 행동 → 몬스터 반격 순으로 진행됩니다. 몬스터를 처치하면 EXP와 확률적으로 물약을 획득합니다.

커맨드 레퍼런스

Manual
커맨드 유형 효과 특이사항
(start_game) 시스템 게임 초기화 및 1층 시작 재시작도 동일 커맨드
(attack) 공격 보통 공격. 데미지 = ATK − 적 DEF (최소 1) 항상 명중
(power) 공격 강타. 데미지 ≈ 2× ATK − 적 DEF 25% 확률 미스 (rng > 74)
(special) 필살기 천둥 일격. 데미지 = ATK × 3, DEF 완전 무시 전투당 1회 — 적 처치 후 다음 전투에서 충전
(guard) 방어 방어 자세. 이번 턴 받는 피해 50% 감소 공격 행동 없음. 턴 종료 후 자동 해제
(potion) 회복 물약 사용. HP +40 (최대 HP 초과 불가) 물약 0개 시 사용 불가 (적은 반격)
(status) 정보 플레이어·현재 적 스탯 출력 전투 소비 없음
exit 시스템 REPL 종료 —
데미지 공식
보통 공격: max(1, ATK − 적DEF)  ·  강타: max(1, ATK − 적DEF + ATK)  ·  필살기: ATK × 3 (DEF 무시)  ·  방어 시 수령 피해: max(1, 원데미지 / 2)

몬스터 도감

Manual

일반 몬스터(층 1–4)는 5종 중 랜덤 등장. 층이 높을수록 HP·ATK 상승. 5층은 항상 보스.

일반 몬스터 스탯 공식 — HP: 20 + (층×8) + rand(8)  ·  ATK: 5 + 층 + rand(4)  ·  DEF: rand(층+2)

메서드 디스패치 우선순위
obj.slot 접근 시 Executor는 메서드를 먼저 탐색한 뒤 필드에 폴백합니다. Hero가 take_damage를 오버라이드하므로 h.take_damage(raw)는 항상 Hero 버전이 호출됩니다.

Hero.init은 (Super.init "용사" 100 15 5)로 Fighter 필드를 초기화. Hero.take_damage는 방어 처리 후 (Super.take_damage adjusted)를 호출해 실제 HP를 감소.

메서드 목록

메서드그룹설명
take_damage(raw) 오버라이드 is_defending이면 raw를 절반으로 줄인 뒤 Super.take_damage 호출

reset_for_battle() 리셋 special_ready = true, is_defending = false (전투 시작 시)

reset_all() 리셋 모든 스탯·플래그를 초기값으로 복원 (start_game에서 사용)
show_full() UI Lv, HP, ATK, DEF, 물약, 필살기 상태 출력 
-- 방어 오버라이드 핵심 로직
(method take_damage (raw)
  {
    (var adjusted raw)
    (if This.is_defending
      (set! adjusted (max2 1 (/ raw 2))))
    (return (Super.take_damage adjusted))
  })

전역 함수

Code

유틸리티

함수설명비고
mod(a, b) 재귀 나머지 연산 SLIP에 % 연산자 없음. 모듈러스 97로 최대 재귀 ~44회
max2(a, b) / min2(a, b) 2항 최대·최솟값 SLIP + 연산자는 2항만 지원
rng() LCG 의사 난수 0–96 공식: _seed = (31×_seed + 7) mod 97
rand_n(n) 0 ~ n−1 정수 mod(rng(), n)

몬스터 스킬 함수

Enemy.take_turn에서 (e, h) 시그니처로 호출됩니다. h.take_damage(raw)가 Hero의 오버라이드 버전을 디스패치합니다.

함수데미지 공식
enemy_normal_atk(e, h)max(1, e.atk − h.def_v)
enemy_heavy_atk(e, h)max(1, e.atk − h.def_v) × 2
enemy_quick_slash(e, h)기본 데미지의 절반 × 2타 (1타 후 생존 체크)
enemy_heavy_or_normal(e, h)rng > 59 → heavy, else normal
enemy_rally_or_normal(e, h)hp < maxhp/2 AND rng > 39 → heal(20), else normal
enemy_dark_slash(e, h)rng > 49 → heavy, else normal
enemy_boss_turn(e, h)저체력+rng>84 회복(25) / rng>74 heavy / rng>49 slash / normal

턴 흐름

Code

do_action(type) 내부의 실행 순서. 매 커맨드 호출마다 이 흐름이 진행됩니다.

1
게임 활성 체크
game_active가 false이면 안내 메시지 출력 후 즉시 반환.
2
플레이어 행동 실행
action_type에 따라 Hero 메서드 호출.
0=do_normal_attack · 1=do_power_attack · 2=use_potion · 3=do_special · 4=do_guard
3
몬스터 반격
cur_enemy.is_alive()가 true일 때만 cur_enemy.take_turn(hero) 호출. 플레이어 행동으로 몬스터가 사망했다면 반격 없음.
4
턴 상태 리셋
hero.reset_turn()으로 is_defending = false. 이후 상태 출력.
5
플레이어 사망 판정
not hero.is_alive() → game_over, game_active=false. 최종 레벨 출력 후 반환.
6
몬스터 사망 판정
not cur_enemy.is_alive() → EXP 지급·레벨업 체크·물약 드롭 체크 → game_floor+1 → 5층 초과 시 클리어, 아니면 spawn_enemy·show_floor_header.
설계 포인트 — Enemy 인스턴스를 매번 새로 생성하지 않고 set-field!로 필드만 교체하는 이유: SLIP에서 인스턴스 재생성 후 전역 변수 참조를 갱신하려면 새 var 선언이 필요한데, SLIP의 var는 재선언을 허용하지 않습니다.
